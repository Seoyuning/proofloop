# ProofLoop

> **"숙제 해줘" → AI가 거절하고, 오히려 학생을 시험 본다.**

학생이 AI에게 과제를 시키면, AI가 답 대신 질문을 던집니다.
학생이 대답하면, AI가 그 답을 채점합니다.
그리고 그 데이터가 실시간으로 교사에게 흘러갑니다.

**AI를 쓸수록 학습이 일어나고, 그 데이터가 교사의 수업이 되는 구조.**
이것이 ProofLoop입니다.

---

## 왜 필요한가

| 현재 교육 현장의 문제 | ProofLoop의 해결 |
|---|---|
| 학생이 AI로 과제를 대행한다 | AI가 답을 거절하고 학생을 역으로 검증한다 |
| 일반 질문에도 AI가 그냥 답만 준다 | **매 답변마다** 이해도 측정 + 서술형 후속 질문이 따라온다 |
| 교사는 학생이 뭘 모르는지 감으로 파악한다 | 학생별 약점과 반 전체 오개념이 데이터로 실시간 집계된다 |
| 수업 자료를 만들 때 학생 데이터가 없다 | 학생이 실제로 틀린 지점에 정렬된 수업 자료/시험을 자동 생성한다 |

---

## 핵심: 지속적 학습 검증 루프

학생은 AI를 자유롭게 씁니다. 하지만 **모든 대화에서, 매번, 예외 없이** AI가 세 가지를 동시에 수행합니다:

```
학생 질문 → AI 답변
              ├─ 교과서 근거 (단원명 + 쪽수)
              ├─ 이해도 측정 (1~5점, DB 저장)
              └─ 서술형 후속 질문 (찍어서 맞출 수 없는 형태)
```

과제를 시키면? AI가 답 대신 개념을 설명하고, 학생에게 서술형 질문을 냅니다.
학생이 답하면? AI가 그 답변을 분석해 이해도를 다시 측정합니다.

**진짜 이해 없이는 빠져나갈 수 없는 구조.**

---

## 학생이 보는 화면

- 가입 시 학년 선택 (중1~고3)
- 교사가 보낸 6자리 초대 코드로 반에 참여
- 반의 교과서에 맞는 **국내 5사 AI 챗봇**이 자동 활성화
- 선생님이 올린 학습자료가 있으면 그 자료를 검색해 근거로 답변(RAG) — **실제 자료 원문 스니펫**을 함께 표시
- 매 답변에 근거 카드 + 이해도 바 + 후속 질문 카드가 표시
- "이 문제 풀어줘" → AI가 거절하고 확인 질문 발행
- 이해도가 오르면 **약점 개념으로 맞춤 연습문제 출제 → AI 채점·피드백** (학년별 문제 데이터 기반 few-shot 생성)

## 교사가 보는 화면

- **반 만들기**: 학년/과목/교과서 지정, 6자리 초대 코드 자동 생성 (클릭하면 복사)
- **실시간 대시보드**: 누적 질문 수, 참여 학생 수, 상위 오개념, 평균 이해도
- **학생별 약점 트래킹**: 학생 한 명 한 명의 약점 개념, 오개념 태그, 최근 질문
- **수업 자료 생성**: 학생 데이터에 정렬된 슬라이드 아웃라인 + 확인 질문 + 교사 메모
- **시험 초안 생성**: 4지선다 문항 + 정답 + 근거 + 예상 함정
- **학생 연습 데이터**: 학생에게 출제된 맞춤 연습문제와 정답률을 보고 시험·복습 자료로 활용
- **내보내기**: 클립보드 복사 / 파일 다운로드

## 55종 실제 한국 교과서 탑재

비상교육, 미래엔, 천재교육, 동아출판, 교학사, 좋은책신사고, YBM 등 10개 출판사.
수학, 국어, 영어, 과학, 사회, 한국사, 물리학, 화학, 생명과학, 지구과학, 정보 등 15개 과목.
교사가 반을 만들 때 교과서를 지정하면, 학생 챗봇이 그 범위 안에서만 동작합니다.

## 기술 스택

| 영역 | 기술 |
|---|---|
| 프론트엔드 | Next.js 16, React 19, TypeScript, Tailwind CSS 4 |
| 백엔드/DB | Supabase (PostgreSQL, Auth, RLS, pgvector) |
| AI 모델 | 국내 5사 오케스트레이션 — LG EXAONE(FriendliAI) · Upstage Solar · KT Mi:dm · SKT A.X K1 · NC VARCO (SKT 라우터가 입력 유형별로 분배, 미발급 모델은 폴백) |
| RAG | Upstage 임베딩 + pgvector — 업로드 학습자료를 검색해 근거 스니펫과 함께 답변 |
| 문서 OCR | Upstage Document Parse (자료/사진 텍스트·수식 추출) |
| 개발 에이전트 | Claude Code (기획~구현~문서화) |
| 폰트/UI | Pretendard, 반응형, 모바일 햄버거 메뉴, 한국어 줄바꿈 최적화 |

## 페이지 구조

| 경로 | 설명 |
|---|---|
| `/` | 스플래시 랜딩 → `/studio/login` 리다이렉트 |
| `/studio/login` | 로그인/회원가입 (역할 + 학년 선택) |
| `/studio/chat` | 학생: 교과서 챗봇 (검증 루프) |
| `/studio/analysis` | 교사: 질문 분석 대시보드 |
| `/studio/generate` | 교사: 수업 자료/시험 생성 + 내보내기 |
| `/studio/classes` | 교사: 반 관리 + 학생별 약점 |
| `/studio/mypage` | 프로필 관리 |

## 로컬 개발

```bash
npm install
npm run dev
```

### 환경변수 (.env.local)

```
NEXT_PUBLIC_SUPABASE_URL=<Supabase 프로젝트 URL>
NEXT_PUBLIC_SUPABASE_ANON_KEY=<Supabase anon key>
GEMINI_API_KEY=<Gemini API 키>
```

### 빌드 확인

```bash
npm run lint
npm run build
```
